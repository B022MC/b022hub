# b022hub Local Purchase Integration Guide

This project currently exposes the purchase page as an embedded iframe, not as a built-in payment API service.

## How it works

1. Enable `Recharge / Subscription` in admin settings.
2. Fill `purchase_subscription_url` with an absolute `http(s)` URL.
3. The user-side purchase page is rendered inside `/purchase`.
4. The same page also provides an `Open in new tab` button as a fallback.

## Query parameters automatically appended by b022hub

When `/purchase` loads your configured URL, the frontend appends these query parameters:

- `user_id`: current logged-in user ID
- `token`: current auth token
- `theme`: `light` or `dark`
- `lang`: current locale such as `zh` or `en`
- `ui_mode`: always `embedded`
- `src_host`: current b022hub origin
- `src_url`: current page URL

## Example

Base URL:

```text
https://billing.example.com/purchase
```

Embedded URL generated by b022hub:

```text
https://billing.example.com/purchase?user_id=12&token=jwt.xxx&theme=dark&lang=zh&ui_mode=embedded&src_host=https%3A%2F%2Fhub.example.com&src_url=https%3A%2F%2Fhub.example.com%2Fpurchase
```

## Integration requirements

- The target page must allow iframe embedding.
- Do not send `X-Frame-Options: DENY`.
- If you use CSP, ensure `frame-ancestors` allows the b022hub domain.
- If embedding is blocked, users can still open the same page in a new tab.

## Theme and locale

- Use the `theme` query parameter to keep your purchase page visually aligned with the current b022hub theme.
- Use the `lang` query parameter to switch copy for Chinese and English users.

## Authentication note

- The `token` query parameter is forwarded from the current logged-in session.
- Only trust it after server-side verification in your own purchase service.
- Avoid relying on the iframe page alone for permission checks.

## Scope

This guide only documents the embedded purchase page behavior already implemented in the current project.
